Subject Patterns

Regex trên subject để xác định outcome. Match theo thứ tự priority, lấy pattern đầu tiên khớp.

APPROVED Review of your (?<app_name>.+) submission is complete\.
priority 1
REJECTED There's an issue with your (?<app_name>.+) submission\.
priority 2
IN_REVIEW Your (?<app_name>.+) status has changed to (In Review|Waiting for Review)
priority 3

Types

Body keyword để detect Type. Payload regex extract metadata từ body (version, event ID, page ID…). Ticket gom theo (app + type + platform).

APP · active · 82 tickets last 30d
Body keyword App Version Payload regex App Version\n(?<version>[\d.]+) for (?<os>\w+) Example match → version=2.4.1 os=iOS
IN-APP EVENT · active · 34 tickets last 30d
Body keyword In-App Events Payload regex In-App Events\n(?<event_name>.+?)\s+(?<event_id>\d+) Example match → event_name=WR Patch 6B event_id=6761699122
CUSTOM PAGE · active · 18 tickets last 30d
Body keyword Custom Product Pages Payload regex Custom Product Pages\n(?<page_name>.+?)\s+(?<page_id>[a-f0-9-]{36}) Example match → page_name=3KF - Set 2: Recall - 03/2026
