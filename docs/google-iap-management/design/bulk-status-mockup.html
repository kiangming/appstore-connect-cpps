Q-BULK locks (kickoff defaults)

  • • Item source: all matching status (cap 1000)
  • • Inactive confirm: count-only display
  • • Progress: spinner (no SSE)
  • • Result: modal + list refresh
  • • Failed: error message per item
  • • Naming: "Bulk Activate" / "Bulk Deactivate"

Architecture pivot — Google ≠ Apple

  • • Google has cross-product batch endpoint (productId="-" wildcard)
  • • 1 POST per ≤100-item chunk (existing helper: newCrossProductBatchActivate)
  • • Sequential chunks — no per-item concurrency machinery
  • • 1000 items ≈ 10 batches ≈ 30-50s total
  • • Legacy fallback per-item w/ concurrency 2 + retry (rare)

Implementation architecture (post-mockup-approval)

Backend

  • • lib/google-iap-management/orchestration/bulk-status.ts (NEW)
  • • Reuses existing newCrossProductBatchActivate() in publisher-client.ts:675
  • • Chunk at 100 items per batch (Google batch quota guidance)
  • • Sequential chunks; per-item drift recovered via legacy fallback path
  • • Cache update: re-fetch via listInAppProducts + upsert into iaps table (overlay desired state — Hotfix 12 pattern)
  • • Two new ActionTypes: BULK_ACTIVATE, BULK_DEACTIVATE

API + UI

  • • POST /api/google-iap-management/apps/[packageName]/iaps/bulk-activate
  • • POST /api/google-iap-management/apps/[packageName]/iaps/bulk-deactivate
  • • Zod schema: 1-1000 SKUs per request
  • • components/google-iap-management/iap-list/BulkStatusModal.tsx — single component, mode prop
  • • Wired in IapListClient.tsx button row (left-of-existing)
  • • router.refresh() on Close so list re-reads cache

Rate-limit handling (Manager directive)

Google's batchUpdateStates is multi-product native — 1 POST per ≤100-item chunk. 1000 items ≈ 10 sequential batches ≈ 30-50s wall time. No per-item concurrency machinery needed (Apple Cycle 40 Phase A pattern was per-IAP and demanded concurrency 2 + retry; Google's batch endpoint dodges that entirely). Legacy fallback path (inappproducts.patch per-item) does inherit withConcurrency(2) + withRetry for the rare app-not-on-new-API case.
