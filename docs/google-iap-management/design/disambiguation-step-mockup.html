Scope symmetry

Same UX for both template scopes

Pricing source Disambiguation applies? Tier column rendering
DEFAULT_TEMPLATE ✅ Yes — GLOBAL scope candidate lookup Read-only when ≤1 candidate; dropdown when >1
APP_TEMPLATE ✅ Yes — APP scope candidate lookup Read-only when ≤1 candidate; dropdown when >1
GOOGLE_DEFAULT ❌ No — auto-bootstrap path, no template lookup "Auto-converted from USD" for every row

Wizard Step 1 (Pricing source) determines which scope is queried; the Preview step's rendering rules don't branch on scope. Manager workflow is identical regardless of source selection.
